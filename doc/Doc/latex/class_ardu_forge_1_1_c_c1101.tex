\doxysubsection{Detailed Description}
This class implements data transmitting and receiving for devices with a \mbox{\hyperlink{class_ardu_forge_1_1_c_c1101}{C\+C1101}} transceiver chip. 

You are free to configure the pins of the device on any available digital pins. Specify the pin assignment in the \mbox{\hyperlink{class_ardu_forge_1_1_c_c1101_a2816afc3b1ec4d3b2c9ef2afa6ce5968}{C\+C1101\+::begin}} method. The implementation has been inspired by Space\+Teddy\textquotesingle{}s (\href{https://github.com/SpaceTeddy/CC1101}{\texttt{ https\+://github.\+com/\+Space\+Teddy/\+C\+C1101}}) and simonmonk\textquotesingle{}s (\href{https://github.com/simonmonk/CC1101_arduino}{\texttt{ https\+://github.\+com/simonmonk/\+C\+C1101\+\_\+arduino}}) \mbox{\hyperlink{class_ardu_forge_1_1_c_c1101}{C\+C1101}} implementations. They where a great help so thanks for sharing. \begin{DoxyWarning}{Warning}
Don\textquotesingle{}t forget that the \mbox{\hyperlink{class_ardu_forge_1_1_c_c1101}{C\+C1101}} requires 3.\+3V input and you will nee a a logic converter or voltage dividers in case you connect from a higher voltage device such as the Arduino. There are tutorials mentioning that it is possible to connect directly, but I fried one breakout board with direct connection, so better be save than sorry. You can use 220ohm/470ohm voltage dividers to connect from a 5V device. 
\end{DoxyWarning}
\begin{DoxyRemark}{Remarks}
The class was mainly tested using 433 M\+Hz. Only small package sizes are allowed since larger package sizes require elaborate syncing of sender and receiver. I was able to send 59 bytes of payload data in single packages without problems.  
\end{DoxyRemark}
